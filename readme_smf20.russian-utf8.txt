Allowed Email Domains 1.5.2
===========================

Мод ограничивает регистрацию пользователей и смену email в профиле по почтовым доменам.

Мод работает только через интеграционные хуки SMF и не изменяет исходный код форума.

Страница настроек:
Админка -> Конфигурация -> Разрешённые почтовые домены

Основные настройки:
1. Включить ограничения почтовых доменов.
2. Разрешённые почтовые домены: по одному полному домену на строку, например example.com.
3. Разрешённые домены первого уровня: по одному домену первого уровня на строку, например ru, com, org.
4. Запрещённые почтовые домены: по одному полному домену на строку, например badmail.com.
5. Запрещённые домены первого уровня: по одному домену первого уровня на строку, например xyz, top, click.
6. Включить логирование попыток регистрации и смены email.

Запрещающие правила имеют приоритет над разрешающими.

Пример: если домен первого уровня com разрешён, адрес user@badmail.com всё равно можно заблокировать, добавив badmail.com в список запрещённых доменов.

На странице настроек есть встроенная проверка email. Результат выводится под полем ввода и показывает, почему адрес разрешён или заблокирован.


Если включено логирование, попытки регистрации и смены email пользователем в своём профиле записываются в стандартный лог ошибок форума с типом aed_email. Для просмотра используйте лог ошибок с фильтром "Разрешённые почтовые домены".

Совместимость этой ветки пакета:
- SMF 2.0.x
- PHP 5.4+
